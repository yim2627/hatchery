# 기능 추가

새로운 기능을 구현하거나 기존 플로우를 확장할 때.

## 진행 순서

1. 영향 받는 영역의 기존 코드를 먼저 읽는다.
2. 최소한의 유저 가시적 동작 변경을 정의한다.
3. 레이어 순서대로 구현한다: 도메인 → 데이터 → 프레젠테이션 → UI.
4. 가장 작고 리뷰 가능한 diff로 구현한다.
5. 비자명한 로직에 테스트를 추가한다.
6. 빌드·테스트를 실행한다: `{{BUILD_COMMAND}}` / `{{TEST_COMMAND}}`

## 확인할 것

- 새 동작이 의도한 UI 플로우에서 도달 가능한가?
- 작업과 무관한 변경이 섞이지 않았는가?
- 비동기 작업의 취소·재진입·상태 경쟁을 고려했는가?
- {{ARCHITECTURE_STYLE}} 패턴을 따르고 있는가?

## 완료 시 정리

- 변경 파일, 동작 요약, 테스트 결과, 리스크.
