# 코드 리뷰

코드, 아키텍처, PR을 리뷰할 때.

## 진행 순서

1. 대상 코드 또는 diff를 읽는다.
2. `.hatchery/skills/`의 활성 스킬 규칙과 대조한다.
3. 발견 사항을 심각도별로 분류한다:
   - **CRITICAL** — 반드시 수정. 정확성·보안 문제.
   - **WARNING** — 높은 확률의 버그 또는 유지보수 리스크.
   - **INFO** — 개선 권장.
4. 각 항목에 파일/라인, 문제, 영향, 수정 방향을 포함한다.

## 완료 시 정리

- 심각도별 그룹화된 발견 사항, 전체 평가, 우선순위 제안.
