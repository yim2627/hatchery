# 버그 수정

동작 오류, 크래시, 상태 전이 문제, 회귀를 고칠 때.

## 진행 순서

1. 버그를 한 문장으로 정리한다.
2. 실패하는 레이어 또는 경계를 먼저 특정한다.
3. 근본 원인 가설을 확인한 후에 패치한다. 증상만 가리지 않는다.
4. 가능하면 회귀 테스트를 추가한다.
5. 빌드·테스트를 실행한다.

## 확인할 것

- 버그가 재현되지 않는가?
- 수정으로 인해 다른 동작이 변하지 않았는가?
- 비동기/상태 사이드 이펙트를 점검했는가?

## 완료 시 정리

- 근본 원인, 수정 내용, 변경 파일, 테스트 결과, 남은 리스크.
