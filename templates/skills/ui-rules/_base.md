# UI 규칙

화면, 컴포넌트, UI 상태와 관련된 작업에 이 스킬을 적용한다.

## 핵심 원칙

- UI 컴포넌트는 레이아웃과 바인딩을 주로 기술해야 한다.
- UI 렌더링 코드에 비즈니스 판단을 직접 넣지 않는다.
- 컴포넌트가 복잡해지면 서브 컴포넌트나 computed property로 추출한다.
- 프레젠테이션 상태를 명시적으로 유지한다.
- loading, success, empty, error 상태를 필요한 곳에서 처리한다.
- 렌더링 경로에서 무거운 작업을 반복 트리거하지 않는다.
- 기존 네비게이션·프레젠테이션 패턴을 따른다.
